[SINTAXIS_Y_LÓGICA_NÚCLEO]

MOD 1: VARIABLES (Cajas de Datos)

Usa const para nombres que no cambian y let para valores que sí (como energía).

const NOMBRE = 'Hércules';
let energia = 100;

MOD 2: CONDICIONALES (Interruptores)

Como un guardia: Deja pasar solo si se cumple la condición.

if (energia > 50) {
  console.log('SISTEMA_OK');
} else {
  console.log('ALERTA');
}

MOD 3: BUCLE FOR (Contador)

Úsalo cuando sepas exactamente cuántas veces quieres repetir algo (ej. 5 veces).

for (let i=0; i < 5; i++) {
  console.log('Escaneando...');
}

MOD 4: BUCLE WHILE (Condición)

Úsalo cuando no sepas el número exacto, sino que dependas de una condición (ej. mientras haya energía).

while (energia > 0) {
  descender(); // Se repite hasta que la condición sea falsa.
}

MOD 5: FUNCIONES (Pequeños Robots)

Paquetes de órdenes que guardas para usar después.

// ROBOT ESTÁNDAR (NOMBRADA):
function activarEscudo() { ... }

// ROBOT RÁPIDO (FLECHA):
const alertar = () => { ... };
[ESTRUCTURAS_Y_DATA]

MOD 6: ARRAYS (Listas de Inventario)

Una lista de cosas guardadas por posición (empezando en 0).

const armas = ['Láser', 'Misil'];
console.log(armas[0]); // 'Láser'

MOD 7: OBJETOS (Planos Técnicos)

Un conjunto de datos con etiquetas (propiedades).

const nave = { motor: 'Warp', hp: 100 };
console.log(nave.motor); // 'Warp'

MOD 8: MÉTODOS PRO (Filtros de Datos)

Herramientas modernas para limpiar tus listas.

const vivos = crew.filter(c => c.hp > 0);

MOD 9: DESESTRUCTURACIÓN (Extracción)

Saca solo lo que necesitas de un objeto gigante.

const { lat, lon } = gps_data;
[INTERACCIÓN_STARK_OS]

MOD 10: SELECTORES (Brazos Robóticos)

Conecta tu código con partes reales de la pantalla.

const fuel = document.getElementById('hud-fuel');

MOD 11: EVENTOS (Sensores de Click)

Detecta cuando el usuario toca un botón o tecla.

btn.addEventListener('click', () => { ... });

MOD 12: ESTILOS (Personalización)

Cambia el aspecto visual de un componente al instante.

box.style.backgroundColor = 'red';

MOD 13: CONTENIDO (Texto Biométrico)

Escribe mensajes directamente en la pantalla HUD.

visor.innerText = '¡BIENVENIDO!';
[PROTOCOLOS_DE_TIEMPO]

MOD 14: TEMPORIZADORES (Relojes)

Haz que algo pase después de unos segundos.

setTimeout(() => { alert('¡BOOM!'); }, 2000);

MOD 15: PROMESAS (Pactos Digitales)

Algo que pasará en el futuro (si todo va bien).

fetch(api).then(res => res.json());

MOD 16: ASYNC/AWAIT (Espera Activa)

Maneja el tiempo como si fuera una lista de tareas lineal.

const data = await getSectorState();
[PROGRAMACIÓN_ORIENTADA_A_OBJETOS]

MOD 22: CONSTRUCTORES (Plantillas)

Usa funciones para crear muchos objetos iguales con this.

function Dron(id) { this.id = id; }
const d1 = new Dron(101);

MOD 23: CLASES (Modernidad)

La forma oficial de agrupar datos y funciones (métodos).

class Dron { constructor(id) { this.id = id; } operar() { ... } }

MOD 24: HERENCIA (Evolución)

Crea una clase basada en otra usando extends y super.

class SuperDron extends Dron { constructor(id) { super(id); } }
[BLINDAJE_Y_APIS_PRO]

MOD 17: TYPESCRIPT (Blindaje de Datos)

Evita que entre "basura" en tus variables fijando su tipo.

let id: string = 'SEC-01'; // Solo texto.

MOD 18: INTERFACES (Contratos)

Define exactamente cómo debe verse un objeto de datos.

interface Pilot { name: string; rank: number; }

MOD 19: GENÉRICOS (Adaptadores)

Funciones que funcionan con cualquier tipo de dato.

function log(val: T) { console.log(val); }

MOD 20: STORAGE (Caja Negra)

Guarda información que no se borre al cerrar la página.

localStorage.setItem('score', '100');

MOD 21: OBSERVERS (Vigilantes)

Detecta cambios en la pantalla automáticamente.

new MutationObserver(callback).observe(node);
