MOD 1: VARIABLES Y MEMORIA

JS

Declaración de Energía

Para guardar datos en la memoria de la nave, usamos tres protocolos de comando con diferentes niveles de seguridad y alcance.

  • const: Valor inamovible (Lectura única). Ideal para IDs de naves o leyes físicas. No permite reasignarse.
  • let: Variable con alcance de bloque. Se puede reasignar (ej: nivel de oxígeno que baja). Es la opción estándar segura.
  • var: Protocolo antiguo. Tiene alcance global/función, lo que puede causar "fugas de datos" (bugs) accidentales.
Ejemplo Novato 
// Lo que no cambia
const NAVE_TIPO = "Estelar";

// Lo que fluye
let municion = 500;
municion = 480; // OK

MOD 2: SELECTORES Y CABLES

JS

Pinchado de Sistemas

Para hablar con una parte visual de la cabina (DOM), debemos seleccionarla. Es como conectar un cable a un puerto específico.

  • getElementById('id'): Selecciona la pieza exacta por su nombre único. Es el más rápido y directo.
  • querySelector('.clase'): Selecciona usando estilos. El punto (.) indica clase, el hashtag (#) indica ID.
  • querySelectorAll('tag'): Trae TODOS los elementos que coincidan en una lista (Array).
Ejemplo Novato: ID 
const fuelBar = document.getElementById('hud-fuel');
Ejemplo Novato: Clase 
const stats = document.querySelector('.hud-stat');

Herramientas de Mando

Una vez seleccionado un sistema, puedes modificar sus propiedades reales:

  • innerText: Cambia el texto visible sin afectar la estructura HTML.
  • style.prop: Cambia propiedades CSS (ej: color, left, opacity).
  • classList.add('cls'): Activa una clase predefinida (ej: activar una animación).
Ejemplo Novato: Cambio 
fuelBar.innerText = "PELIGRO: 10%";
fuelBar.style.color = "red";

MOD 3: MOTORES Y ACCIONES

JS

Motores Maestros

Las funciones agrupan órdenes. Se pueden llamar múltiples veces para realizar la misma tarea.

  • function f() {}: Declaración estándar. Tiene su propio contexto 'this'.
  • const f = () => {}: Función de flecha. Más corta y ágil, ideal para cálculos rápidos.
Ejemplo Novato: Función 
function disparar() {
  log("Laser activado");
}
const turbo = () => log("Turbo OK");

Callbacks (Órdenes de Espera)

Un Callback es una función que pasas a OTRA función como si fuera una pieza de repuesto. Se ejecuta cuando la primera termina.

Ejemplo Novato: Callback 
// Al pulsar (click), ejecuta la función
boton.onclick = function() {
  log("Boton pulsado");
};

MOD 4: SISTEMAS DE DATOS

JS

Sensores y Listas (Arrays)

Estructuras para guardar muchos datos ordenados por posición (índice).

  • push(x) / pop(): Añade o quita piezas al final de la lista.
  • forEach(f): Lee cada pieza de la lista y le da una orden.
  • map(f): Transforma toda la lista en una nueva versión.
Ejemplo Novato: Arrays 
let naves = ["Alpha", "Beta"];
naves.push("Gamma");
naves.forEach(n => log("Detectada: " + n));

Planos Técnicos (Objetos)

Guardan datos con nombres (clave:valor). Ideales para describir el estado de la nave.

Ejemplo Novato: Objetos 
let telemetria = {
  hp: 80,
  motor: "Encendido"
};
// Extraer datos rápido
let { hp } = telemetria;

MOD 5: COMUNICACIÓN ESTELAR (ASYNC)

JS

Espera Asincrónica

Controla procesos que tardan en completarse (ej: cargar combustible, esperar respuesta de base).

  • setTimeout(f, ms): Ejecuta una orden tras X milisegundos.
  • Promise: Un ticket que promete darte un resultado en el futuro.
  • async/await: Sincroniza visualmente el código para que "espere" sin bloquear la nave.
Ejemplo Novato: Async 
async function cargarSistemas() {
  log("Iniciando carga...");
  await esperar(1000); 
  log("¡Carga completa!");
}

MOD 6: PROTOCOLOS ÉLITE (TS)

TS

Seguridad de Datos

TypeScript asegura que cada sensor solo reciba el tipo de dato correcto (número con número, texto con texto).

  • let hp: number: Obliga a que hp sea siempre numérico.
  • interface IShip: Un contrato estricto sobre qué datos debe tener una nave.
  • enum State: Un set fijo de opciones (ej: ON, OFF, ERROR).
Ejemplo Novato: TS 
interface Piloto {
  id: number;
  nombre: string;
}
let p1: Piloto = { id: 1, nombre: "Stark" };

MOD 7: CONTROL DE DAÑOS (ERRORES)

JS

Blindaje (Try/Catch)

Envuelve partes peligrosas del código. Si algo falla dentro, el error se captura y la nave sigue volando en lugar de colapsar.

Ejemplo Novato: Error 
try {
  conectarSensorFallido();
} catch (e) {
  log("FALLO EVITADO: " + e.message);
} finally {
  log("Reestableciendo sistemas...");
}
